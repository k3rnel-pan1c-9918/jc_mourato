Requisitos

Propõem-se a conceção de uma aplicação web asp.net de um sistema de informação para uma oficina de automóveis.

A referida aplicação deverá fazer uso de:

ASP.NET Core MVC com autenticação

Padrão repository;

Entity Framework Core

Base de Dados em SQL Server

Github Repository

C R U D


Sistema completo de autenticação:

login && logout

registo de novos utilizadores

recuperação de password por email

...


Criação de Roles com um mínimo de 4 tipos de utilizadores

admin

staff

client

anon


A aplicação deverá estar funcional e online quando for testada e apresentada

A aplicação não pode em caso algum rebentar nem mostrar os ecrãs de erros de desenvolvimento, devendo criar todas as views para a gestão de erros e conflitos dos CRUD’s.

Sistema de front-end original e adaptado ao projeto.

Funcionaidades Especificas

Implementação de módulos:

Veículos

Clientes

Mecânicos

Reparação

Faturação

Marcações


Deverá ser também implementado um módulo de Comunicações com as seguintes funcionalidades:

 • Enviar email para todos os clientes, avisando por exemplo, que a oficina vai estar fechada;

 • Enviar email a relembrar marcação para o dia seguinte;

 • Enviar email a desmarcar marcações de um determinado mecânico.


O administrador do sistema apenas gere os restantes utilizadores (funcionários e clientes), criando-lhes as contas para estes posteriormente poderem entrar na plataforma.

Irá também disponibilizar as diferentes especialidades de cada mecânico bem como das diferentes marcas e modelos de Viaturas.

Atenção que no processo de criação da conta, qualquer utilizador criado no ponto anterior, irá receber primeiro um email onde deverá alterar a sua password.

Deverá também haver um funcionário (rececionista), que irá marcar as reparações mediante a disponibilidade de horários.

Finalmente irão existir os clientes e os seus veículos, onde estes poderão efetuar todo o tipo de consultas referente ao histórico das suas reparações por viatura.

Os utilizadores anónimos apenas podem consultar os mecânicos e ofertas disponíveis.

Outros aspectos

Todos os utilizadores, com exceção dos anónimos, deverão ter a possibilidade de ter foto de perfil.

O funcionário, quando se autentica, deverá ser direcionado para um dashboard onde aparecem os alertas lançados.

Todos os veículos deverão ter um histórico de com todas as intervenções que pode ser consultado online pelo cliente.

Deverá haver o respeito por não apagar e atualizar em cascata, sendo necessária a comunicação entre os diversos utilizadores.

Deverá também ser criada e publicada uma web API que envie todas as intervenções de um determinado veículo.
